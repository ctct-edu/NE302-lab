# 在庫管理システム 要件定義書

## 概要

小規模店舗向けの在庫管理システム。商品の登録・検索、在庫数の管理、在庫アラート機能を提供する。

## 機能要件

### MUST（必須機能）

| 機能         | 説明                                                         |
| ------------ | ------------------------------------------------------------ |
| 商品登録     | 商品名、カテゴリ、単価を登録できる                           |
| 商品一覧     | 登録済み商品の一覧を表示できる                               |
| 在庫管理     | 入庫・出庫を記録し、現在の在庫数を管理できる                 |
| 在庫アラート | 在庫が閾値以下になった商品を警告表示できる                   |

### WANT（あれば良い機能）

| 機能         | 説明                                       |
| ------------ | ------------------------------------------ |
| 検索機能     | 商品名、カテゴリで絞り込み検索ができる     |
| レポート出力 | 在庫状況を CSV ファイルとして出力できる    |

## ユースケース

### UC-1: 商品を登録する

- **アクター**: 店長
- **事前条件**: ログイン済み
- **基本フロー**:
  1. 商品登録画面を開く
  2. 商品名、カテゴリ、単価を入力
  3. 登録ボタンを押す
  4. 登録完了メッセージが表示される
- **代替フロー**:
  - 商品名が空の場合 → エラーメッセージを表示
  - 単価が負数の場合 → エラーメッセージを表示

### UC-2: 在庫を入庫する

- **アクター**: 店長
- **事前条件**: 商品が登録済み
- **基本フロー**:
  1. 商品を選択
  2. 入庫数量を入力
  3. 入庫ボタンを押す
  4. 在庫数が増加し、履歴に記録される
- **代替フロー**:
  - 数量が 0 以下の場合 → エラーメッセージを表示

### UC-3: 在庫を出庫する

- **アクター**: 店長
- **事前条件**: 商品が登録済み、在庫が存在
- **基本フロー**:
  1. 商品を選択
  2. 出庫数量を入力
  3. 出庫ボタンを押す
  4. 在庫数が減少し、履歴に記録される
- **代替フロー**:
  - 在庫が不足している場合 → エラーメッセージを表示

### UC-4: 在庫アラートを確認する

- **アクター**: 店長
- **事前条件**: 在庫情報が存在
- **基本フロー**:
  1. 在庫アラート画面を開く
  2. 閾値以下の商品一覧が表示される
- **代替フロー**:
  - 該当商品がない場合 → 「アラート対象なし」と表示

## 非機能要件

- レスポンス時間: 3 秒以内
- 同時接続: 10 ユーザー程度
- データ保持: 1 年間
