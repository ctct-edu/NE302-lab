# 在庫管理システム 要件定義書

## 概要

小規模店舗向けの在庫管理システム。商品の登録・管理と在庫数の追跡を行う。

## 機能要件

### MUST（必須機能）

| 機能         | 説明                                       |
| ------------ | ------------------------------------------ |
| 商品登録     | 商品名、カテゴリ、単価を登録できる         |
| 商品一覧     | 登録済み商品の一覧を表示できる             |
| 商品検索     | 商品名、カテゴリで絞り込みができる         |
| 在庫管理     | 入庫・出庫の記録ができる                   |
| 在庫アラート | 在庫が閾値以下になったら警告を表示する     |

### WANT（あれば良い機能）

| 機能           | 説明                           |
| -------------- | ------------------------------ |
| レポート出力   | 在庫状況を CSV で出力できる    |
| 発注管理       | 発注の作成・管理ができる       |

## ユースケース

### UC-01: 商品を登録する

- **アクター**: 店長
- **事前条件**: なし
- **基本フロー**:
  1. 商品登録画面を開く
  2. 商品名、カテゴリ、単価を入力
  3. 登録ボタンを押す
  4. 登録完了メッセージが表示される
- **代替フロー**:
  - 商品名が空の場合 → エラーメッセージを表示
  - 単価が負数の場合 → エラーメッセージを表示

### UC-02: 商品を検索する

- **アクター**: 店長
- **事前条件**: 商品が登録済み
- **基本フロー**:
  1. 検索条件を入力（商品名、カテゴリ）
  2. 検索ボタンを押す
  3. 条件に合う商品が一覧表示される
- **代替フロー**:
  - 該当商品がない場合 → 「該当する商品がありません」を表示

### UC-03: 在庫を更新する

- **アクター**: 店長
- **事前条件**: 商品が登録済み
- **基本フロー**:
  1. 商品を選択
  2. 入庫または出庫を選択
  3. 数量を入力
  4. 更新ボタンを押す
  5. 在庫数が更新される
- **代替フロー**:
  - 出庫数が在庫数を超える場合 → エラーメッセージを表示

### UC-04: 在庫アラートを確認する

- **アクター**: 店長
- **事前条件**: 商品が登録済み、閾値が設定済み
- **基本フロー**:
  1. アラート画面を開く
  2. 閾値以下の商品が一覧表示される
- **代替フロー**:
  - アラート対象がない場合 → 「アラートはありません」を表示
